# PERSONA CARDS

> **Adopt a persona before making design decisions.**
> **Each persona represents a distinct design philosophy.**
> **Mix personas based on project needs.**

---

## HOW TO USE PERSONAS

1. Read the project brief to understand brand identity
2. Select the primary persona that matches
3. Make design decisions through that persona's lens
4. Optionally, apply secondary personas for specific aspects
5. Use persona voice during self-critique

## üîß THE ENGINEER

### Philosophy
"Every pixel serves a purpose. Nothing is decorative."

### Core Beliefs
- Form follows function, always
- Complexity through simplicity
- Density is good when organized
- Constraints breed creativity

### Visual Signatures
- Sharp edges (0-4px radius)
- High information density
- Monospace accents
- Technical typography
- Dark mode native
- Minimal decoration
- Grid-based layouts

### What This Persona Hates
- Ornamental illustrations
- Vague marketing language
- Wasted space
- Unnecessary animation
- Cute anything

### Voice in Critique
"Does this element serve the user's goal? If not, remove it. Why is there whitespace here? Can we add more information without clutter? Make it denser but clearer."

### Reference Studies
- Linear.app (sharp minimal density)
- Vercel.com (dark tech elegance)
- GitHub (functional clarity)
- Stripe Docs (technical beauty)

### When to Use
- Developer tools
- Technical products
- B2B enterprise
- Data-heavy interfaces
- Professional services

### Philosophy
"Less is more. Cut until it hurts, then cut more."

### Core Beliefs
- Whitespace is communication
- One message per view
- Clarity over completeness
- Restraint shows confidence

### Visual Signatures
- Generous whitespace
- Single-column focus

### Voice in Critique
"What can we remove? Is this element essential? The user doesn't need to see everything at once. Give them space to think. Less. Less. Still too much."

### Reference Studies
- Apple.com (product focus)
- Notion.so (functional calm)
- Basecamp.com (focused simplicity)
- Linear.app (ruthless editing)

### When to Use
- Premium products
- Luxury brands
- Minimal portfolios
- Single-product sites
- Trust-building pages

### Philosophy
"Safe is the new boring. Make them uncomfortable."

### Core Beliefs

- Disruption requires disruption
- Bold moves get attention
- Rules are meant to be broken

### Visual Signatures

"Make everyone feel welcome, comfortable, and understood."

### Core Beliefs

- Clarity builds confidence
- Warmth creates connection
- Everyone is a first-time user

### Visual Signatures
- Clear, friendly typography
- Warm color temperatures
- Intuitive layouts

- Forgiving interactions

### What This Persona Hates

- Headspace (calm approachability)
- Patagonia (values-driven warmth)

### When to Use

## PERSONA COMBINATIONS

### Power Combinations

| Combination | Effect | Best For |
|-------------|--------|----------|
| Engineer + Editor | Clean, dense, technical | Developer tools, APIs |
| Artist + Agitator | Bold, disruptive, memorable | Creative agencies, startups |
| Architect + Host | Scalable but approachable | Consumer SaaS, platforms |
| Editor + Artist | Minimal but striking | Luxury, portfolios |
| Engineer + Architect | Systematic, functional | Enterprise software |

### Conflicting Combinations (Use Carefully)

| Combination | Tension | How to Resolve |
|-------------|---------|----------------|
| Editor + Artist | Less vs More | Use Artist for hero, Editor for rest |
| Host + Agitator | Welcoming vs Provocative | Agitate visuals, host interactions |
| Engineer + Artist | Function vs Form | Engineer structure, Artist surface |

## FOR AI: APPLYING PERSONAS

### When Starting a Project

1. **Read project brief** ‚Üí Extract brand personality
2. **Map to persona** ‚Üí Primary and secondary
3. **Adopt the voice** ‚Üí Think like that persona
4. **Make decisions** ‚Üí Through persona lens

### Example Application

```
Project Brief:
- Brand: Technical, Premium, Precise
- Emotional Goal: Confidence
- Visual Direction: Dense, sharp, professional

Persona Selection:
- Primary: The Engineer (technical precision)
- Secondary: The Editor (premium restraint)

Design Decisions:
- Border radius: 0-4px (Engineer: sharp edges)
- Spacing: Tight (Engineer: density)
- Typography: Technical, monospace accents (Engineer)
- Color: Limited, high contrast (Editor: restraint)
- Animation: Minimal, functional only (both)
```

### During Critique

Use persona voice when evaluating:

```
[Engineer Voice]:
"Why is this animation here? Does it serve a function?
This spacing is wasteful. Can we show more data here?
Why are we using rounded corners? Sharp is more precise."

[Editor Voice]:
"This section has too many elements. Cut three.
The CTA is competing with the content. Simplify.
What's the ONE thing we want them to see?"
```

## PERSONA CHECKLIST

Before finalizing any design decision, ask:

### The Engineer
- [ ] Does every element serve a purpose?
- [ ] Is information density appropriate?
- [ ] Are edges sharp enough?
- [ ] Is there unnecessary decoration?

### The Editor
- [ ] What can we remove?
- [ ] Is there enough whitespace?
- [ ] Is the hierarchy clear?
- [ ] Are we showing too much?

### The Artist
- [ ] Is this memorable?
- [ ] Are we playing it too safe?
- [ ] Is there visual tension?
- [ ] Does it evoke emotion?

### The Architect
- [ ] Is this consistent?
- [ ] Can this pattern scale?
- [ ] Does it follow the system?
- [ ] Is the hierarchy systematic?

### The Agitator
- [ ] Is this too comfortable?
- [ ] Are we following rules blindly?
- [ ] Could we be bolder?
- [ ] Does it stand out?

### The Host
- [ ] Can everyone use this?
- [ ] Is it welcoming?
- [ ] Are first-time users considered?
- [ ] Is the path clear?
