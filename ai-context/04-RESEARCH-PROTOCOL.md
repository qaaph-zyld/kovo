# RESEARCH PROTOCOL

> **Research before implementation. Always.**
> **Extract patterns, never copy visuals.**
> **Document findings for future reference.**

---

## WHY RESEARCH MATTERS

Without research, AI defaults to:
- Tailwind defaults
- Common patterns from training data
- Safe, generic structures
- Template-like compositions

Research grounds decisions in real-world success.

## WHAT TO RESEARCH

### Primary Research Targets

1. **Direct Competitors** - Sites solving similar problems
2. **Category Leaders** - Best-in-class in your space
3. **Design Exemplars** - Sites known for great design
4. **User Expectations** - What users in your space expect

### Research Categories

| Category | What to Look For | Why It Matters |
|----------|------------------|----------------|
| Layout | Grid structures, section composition | How information is organized |
| Typography | Scale, contrast, hierarchy | How content is read |
| Color | Palette usage, accent application | How mood is created |
| Spacing | Density, rhythm, breathing | How content feels |
| Interaction | States, transitions, feedback | How users engage |
| Conversion | CTA placement, path design | How action is driven |

---

## ANALYSIS FRAMEWORK

### For Each Reference Site, Document:

#### 1. First Impressions (5 seconds)
- What catches the eye first?
- What's the emotional tone?
- What action is being pushed?

#### 2. Layout Architecture
```
HERO STRUCTURE:
- Content placement: [left/center/right]
- Visual weight distribution: [description]
- CTA prominence: [high/medium/low]

SECTION PATTERN:
- Section count above fold: [number]
- Section rhythm: [consistent/varied]
- Grid structure: [columns/rows/asymmetric]
```

#### 3. Typography Analysis
```
HEADLINE:
- Font: [family]
- Size: [approximate]
- Weight: [light/regular/medium/bold]
- Line height: [tight/normal/loose]
- Letter spacing: [tight/normal/loose]

BODY:
- Font: [family]
- Size: [approximate]
- Weight: [light/regular/medium]
- Line height: [tight/normal/loose]
- Max width: [narrow/medium/wide]

CONTRAST RATIO:
- Headline to body: [X:1]
- Primary to secondary: [description]
```

#### 4. Color Documentation
```
PRIMARY COLOR:
- Hue: [color family]
- Saturation: [vivid/muted]
- Lightness: [dark/medium/light]

ACCENT USAGE:
- Where used: [locations]
- Frequency: [sparse/moderate/heavy]

BACKGROUND:
- Type: [solid/gradient/pattern]
- Color: [description]
```

#### 5. Spacing Patterns
```
DENSITY:
- Overall: [airy/balanced/dense]
- Above fold: [description]
- Section padding: [tight/comfortable/generous]

RHYTHM:
- Consistency: [consistent/varied]
- Vertical rhythm: [tight/loose]
- Horizontal rhythm: [tight/loose]
```

#### 6. Component Patterns
```

- Shape: [sharp/rounded/pill]

- Content density: [minimal/medium/dense]

- Content: [links count, style]
```

#### 7. Interaction Patterns

**Philosophy**: Sharp minimal density

| Element | Pattern | Value |
|---------|---------|-------|

| Typography | Technical | SF Pro, Inter |
| Spacing | Tight | Dense but readable |

| Animation | Functional | Fast, purposeful |

**Extracted Principles**:
- Information density is a feature
- Sharp edges signal precision
- Motion should clarify, not decorate
- Every pixel earns its place

## FOR AI: RESEARCH EXECUTION

### When User Provides References

1. **Visit each reference** (if web search available)
2. **Document using analysis framework**
3. **Extract principles, not visuals**
4. **Apply to project context**

### When User Doesn't Provide References

1. **Ask for 3-5 references** in their niche
2. **If not available**, use pre-analyzed library
3. **Match project type** to closest reference
4. **Document reasoning** for pattern choices
